# 高质量联盟测评页面标准（中文版）

## 目的
本文档定义了什么是"好的"联盟测评页面——一个能在自然搜索中排名、能赚取真实佣金、并在需要时可作为付费广告落地页的页面。

将此标准同时作为**创作指南**和**自检清单**使用。

---

## I. SEO 标准：页面必须获得自然流量

### 1. 搜索意图匹配（关键）
- **信息类查询**（例如："skyrover s1 review"）：页面必须在首 300 字内回答"它好吗？有什么坑？适合谁？"
- **商业类查询**（例如："best mini drone under 250g"）：页面必须包含对比表和明确的胜者推荐。
- **交易类查询**（例如："buy skyrover s1"）：页面必须在首屏展示价格、库存状态和醒目的购买 CTA。

**自检**：如果用户从 Google 搜索 [目标关键词] 后进入此页面，能否在不滚动超过一屏的情况下找到答案？

### 2. E-E-A-T 信号
- **经验**：包含一手观察（"在我们的测试中..."、"飞行 20+ 次后..."）。即使是基于研究的写作，也要写得具体。
- **专业性**：使用正确的技术术语。包含精确数字的参数表。
- **权威性**：链接到官方来源、独立评测（The Verge、Reddit、B&H）和竞品规格。
- **可信度**：包含清晰的免责声明、诚实的缺点（绝不 100% 好评）、以及更新的发布日期。

**自检**：这个页面看起来像是真正用过产品的人写的，还是像复制 Amazon 卖点的人写的？

### 3. On-Page SEO 检查清单
| 元素 | 标准 |
|---|---|
| Title Tag | 包含主关键词 + 角度 + 品牌。少于 60 字符。例如："Skyrover S1 Review: Best Sub-249g Drone Under $300?" |
| Meta Description | 包含主关键词、一个利益点和一个钩子。少于 160 字符。 |
| H1 | 包含主关键词。每页只有一个 H1。 |
| URL Slug | 简短、关键词丰富、连字符分隔。例如：`/review/skyrover-s1-vs-x1/` |
| 图片 Alt Text | 描述性、关键词自然。不要"image1.jpg"。 |
| 内部链接 | 链接到站内 2-3 篇相关评测或指南。 |
| 外部链接 | 链接到 3-5 个权威来源（官方规格、独立评测）。 |
| 结构化数据 | JSON-LD 格式的 Article + Product + Review + FAQPage。 |
| 首 100 词 | 必须包含主关键词和清晰的价值主张。 |
| 内容长度 | 小众产品最少 1,500 词；竞争激烈的词 2,500+。 |
| 目录 | 超过 2,000 词的页面使用。提升 SERP 点击率（跳转链接）。 |

### 4. 技术 SEO
- **页面速度**：LCP < 2.5s，CLS < 0.1，FID < 100ms。Astro 静态站在这方面有优势；注意图片大小。
- **移动优先**：Hero 区域在 375px 宽度下必须显示良好，无横向滚动。
- **Canonical URL**：必须与上线 URL 完全一致。
- **Robots**：必须可索引（`noindex` 仅用于单薄/标签页面）。

## II. 转化标准：页面必须赚钱

### 1. 首屏（前 600px）必须卖货
在首屏内，读者必须看到：

```
[产品图]     [产品名称]
             ⭐ 评分（例如：8.7/10）
             💰 价格（现价 vs 原价）
             ✓ 3 个 bullet-point 卖点
             [🛒 查看价格 / 立即购买] — 联盟链接
             [信任徽章：免费配送、保修、退换政策]
```

**自检**：如果有人点击了 "Skyrover S1 price" 的 Google 广告，他们能否在 2 秒内看到价格和购买按钮？

### 2. "倒金字塔"内容结构
按重要性从高到低排列信息：

1. **首屏摘要**（是什么、适合谁、结论）
2. **快速参数表**（可扫描）
3. **核心问题**（例如："画质真的够 4K 吗？"）
4. **深度章节**（设计、性能、电池等）
5. **对比**（vs 主要竞品）
6. **优缺点**（可快速浏览）
7. **适合谁买**（用户画像匹配）
8. **FAQ**（异议处理）
9. **最终推荐**（按预算分类的表格）
10. **CTA + 免责声明**

**为什么有效**：冲动型买家在 #1-2 转化。研究型买家在 #3-7 转化。比价型买家在 #5 转化。其他人在 #9 转化。

### 3. CTA 密度与位置
| 位置 | CTA 类型 | 目的 |
|---|---|---|
| 首屏 | 主按钮 | 捕获冲动型买家 |
| 第 2 段后 | 内嵌文字链接 | 捕获早期兴趣 |
| 参数表后 | 横幅/按钮 | 捕获参数阅读者 |
| 对比表后 | 按钮 | 捕获比价购物者 |
| "适合谁买"后 | 按钮 | 捕获已决定的读者 |
| 文章末尾 | 主按钮 + 备选 | 捕获所有人 |

**规则**：读者滚动不超过 800px 就必须看到可点击的购买路径。

### 4. 信任与紧迫感元素
- **社会证明**：提及 Amazon 评论数量、综合星级、或"已售出 10,000+ 台"。
- **专家可信度**："TechPick Editor" 署名 + 照片。不要 "admin" 或无作者。
- **价格锚定**：在现价旁边划掉原价。
- **稀缺/紧迫（仅限真实）**："当前优惠截至 [日期]。" 绝不造假。
- **风险逆转**：提及保修、退换政策或退款保证。

### 5. "单一职责"原则
页面的每个章节必须只做一件事：
- **首屏**：阻止滚动并建立相关性。
- **参数表**：回答"它有 X 功能吗？"
- **深度评测**：回答"宣传是真的吗？"
- **对比**：回答"为什么选它而不是另一个？"
- **FAQ**：回答"如果我担心 Y 怎么办？"
- **最终推荐**：回答"具体该买哪个型号/版本？"

如果一个章节没有回答特定的买家问题，删掉它。

## III. 广告就绪标准：页面必须能作为付费落地页

### 1. 信息匹配（广告 → 落地页）
| 广告说 | 落地页必须展示 |
|---|---|
| "Skyrover S1 - $249" | H1 包含 "Skyrover S1"，价格显示 "$249" |
| "40 分钟续航" | 参数表显示 "40 min"，首屏 bullet 提及 |
| "无需 FAA 注册" | FAQ 回答 249g 法规问题 |
| "最适合新手" | "适合谁买"针对新手 |

**Google Ads 质量分影响**：高信息匹配 = 更高质量分 = 更低 CPC。

### 2. 广告加载速度
- **目标**：首屏内容绘制低于 1.5 秒。
- **原因**：Google Ads 对慢落地页惩罚更高 CPC。
- **做法**：图片压缩为 WebP，首屏以下图片懒加载，首屏无第三方脚本。

### 3. 广告移动优先
- 60%+ 的 Google 搜索流量来自移动端。
- 首屏在移动端必须垂直堆叠：图片在上，文字在下，按钮全宽。
- CTA 按钮必须适合拇指点击（最小 44px 高度，高对比度）。

### 4. 无出口泄漏（仅当作为广告落地页时）
- 移除导航菜单或折叠为汉堡菜单。
- 移除侧边栏小部件。
- 移除首屏以上无关的内部链接。
- 让用户专注于 THIS 产品。

**注意**：仅当 actively 向此页面投放广告时适用。对于自然 SEO 流量，完整导航没问题。

### 5. 转化追踪就绪
- 页面必须有干净、可预测的 URL 结构，以便设置 Google Ads 转化目标。
- 联盟链接点击应可作为事件追踪（例如通过 Google Tag Manager 或简单的 `onclick` data layer push）。

## IV. 内容质量门槛："你会把它发给朋友吗？"

### "朋友测试"
发布前，想象把这篇评测发给一位真正在考虑购买此产品的朋友。他们是否会：
- [ ] 感觉理解了产品的真实优缺点？
- [ ] 知道它是否适合*他们*的具体情况？
- [ ] 信任你不是在单纯为品牌站台？
- [ ] 如果准备好了，有清晰、低摩擦的购买路径？

如果任何答案是"否"，在发布前修复。

### 反模式（"坏"页面长什么样）
| 坏模式 | 为什么失败 |
|---|---|
| 100% 好评，无缺点 | 读者不信任。Google 检测到单薄联盟内容。 |
| 复制参数无分析 | 相比官方页面无增值。不会排名。 |
| 无比品对比 | 错过商业搜索意图。转化率更低。 |
| 联盟链接只在底部 | 80% 读者永远滚动不到那里。佣金流失。 |
| 只有 stock photo，无真实图片 | 看起来通用。E-E-A-T 分数下降。 |
| 过时的价格/参数 | 高跳出率。用户快速返回 SERP（坏信号）。 |
| 文字墙，无表格/列表 | 难以扫描。移动端用户跳出。 |

## V. 发布前自检清单

每篇页面上线前使用。

### SEO
- [ ] Title < 60 字符，包含主关键词
- [ ] Meta description < 160 字符，包含关键词 + 钩子
- [ ] H1 包含主关键词，只有一个 H1
- [ ] URL 简短且关键词丰富
- [ ] 首 100 词包含主关键词
- [ ] 至少 1,500 词（竞争激烈的词 2,500+）
- [ ] 包含 2-3 个站内相关内容的内部链接
- [ ] 包含 3-5 个权威来源的外部链接
- [ ] 所有图片有描述性 alt 文本
- [ ] JSON-LD 结构化数据存在

### 转化
- [ ] 首屏展示产品图 + 价格 + CTA 按钮
- [ ] 全页至少 6 个 CTA 触点
- [ ] 优缺点章节存在（最少 4 个优点，3 个缺点）
- [ ] 包含竞品对比表
- [ ] "适合谁买"章节有清晰的用户画像
- [ ] FAQ 章节解答 3+ 常见异议
- [ ] 最终推荐包含按预算分类的选项
- [ ] 联盟链接正确且可用
- [ ] 免责声明存在

### 广告就绪
- [ ] 页面加载低于 2.5 秒（移动端测试）
- [ ] 首屏内容在 375px 宽度下正确堆叠
- [ ] CTA 按钮适合拇指点击且高对比度
- [ ] 信息与计划投放的主要广告角度匹配
- [ ] 无损坏的第三方脚本阻塞渲染

### 技术
- [ ] `npm run build` 0 错误成功
- [ ] 图片存在于 `dist/images/reviews/[品牌]/`
- [ ] 页面存在于 `dist/review/[slug]/index.html`
- [ ] 构建中无 `example.com` URL
- [ ] 分类存在于 `helpers.ts`
- [ ] 作者存在于 `helpers.ts`

## VI. 评分量表（可选）

为每个维度打分 0-10。任何维度低于 6 分的页面在推广前应改进。

| 维度 | 权重 | 10/10 长什么样 |
|---|---|---|
| 搜索意图匹配 | 25% | 页面完美回答带来用户的查询 |
| E-E-A-T | 20% | 感觉像有真实经验的专家写的 |
| 转化设计 | 25% | 每种读者类型都有清晰的购买路径 |
| 内容深度 | 15% | 比 SERP 第一页的竞争页面更深入 |
| 技术质量 | 15% | 快速、移动完美、结构化数据、干净代码 |

**总分目标**：投放广告预算前 8.0+；仅做 organic 时 7.0+ 可接受。

## 总结

"好"的联盟测评页面不只是参数集合加联盟链接。它是：

1. **一个 SEO 资产** — 因为比竞争对手更好地回答搜索意图而获得自然流量。
2. **一个销售页面** — 因为建立信任、处理异议、让购买无摩擦而转化。
3. **一个广告落地页** — 因为加载快、信息匹配、聚焦用户而可随时接入 Google Ads。

最好的页面同时做到这三点。这就是标准。
